2.1 #Mixins.assert assert($description = null)

[public:{output}] Define a CSS-output assertion. Use it as a wrapper around one output() block and one expect(), contains(), or contains-string() check. JavaScript runner output assertions are parsed with PostCSS.

Parameters:

  • $description = null - {string} Optional output assertion description. When omitted, the JavaScript parser uses the current test name for runner output.
  • {block} - Use `output()` and `expect()` mixins to define blocks for comparison

~

Usage:

+test('Stylus math compiles before output') {
  +assert('You can also describe the assertion...') {
    +output() {
      width: 14em + 2;
    }
    +expect() {
      width: 16em;
    }
  }
}

2.12 #Mixins.contains contains($selector)

[public:{output}] Define CSS that must be present within the sibling output() block. The JavaScript runner uses structural PostCSS containment: expected rules, declarations, comments, and at-rules must exist, declaration order inside a matching rule does not matter, and selectors are compared as normalized strings rather than selector algebra.

Parameters:

  • $selector = true - {boolean} Optionally wrap the contents in a `.test-output` selector block, so you can test property-value output directly.
  • {block} - Include any content that are part of this test

~

Usage:

+test('Stylus math compiles before output') {
  +assert() {
    +output() {
      height: 100%;
      width: 14em + 2;
    }
    +contains() {
      width: 16em;
    }
  }
}
